Model Evaluation

Evaluation Metrics

  • Weighted Precision: 0.9992

    Precision measures how accurate the positive predictions are. A precision of 0.9992 means that 99.92% of the cases predicted as positive are actually correct. This suggests that the model rarely makes mistakes when predicting positive outcomes.

  • Weighted Recall: 0.9988

    Recall evaluates the model's ability to identify all relevant cases. A recall of 0.9988 indicates that 99.88% of actual positive cases were correctly identified. This shows the model is very good at finding positive cases, minimizing false negatives.

  • Weighted F1-Score: 0.9990

    The F1-Score is the harmonic mean of precision and recall. A score of 0.9990 reflects a very balanced relationship between precision and recall, meaning the model does a great job at both finding positives and making accurate positive predictions.

  • Accuracy: 0.9988

    Accuracy represents the proportion of correct predictions out of all predictions made. A value of 0.9988 means that 99.88% of the predictions are correct overall, indicating the model performs well across all categories.

  • Macro F1-Score: 0.7125

    The Macro F1-Score calculates the F1-Score independently for each class and then averages them. A score of 0.7125 indicates moderate performance across all classes, and shows that the model might perform better for some classes than others.

  • Micro F1-Score: 0.9988

    The Micro F1-Score aggregates the contributions of all classes to calculate the average F1-Score. A value of 0.9988 indicates excellent overall performance, giving equal importance to all classes combined, regardless of class size.

  • MCC (Matthews Correlation Coefficient): 0.9979

    The Matthews Correlation Coefficient measures the quality of binary classifications. A value of 0.9979 indicates a very strong correlation between predicted and actual classifications, suggesting near-perfect prediction accuracy for binary outcomes.

  • Kappa: 0.9979

    Kappa assesses the agreement between predicted and observed classifications, adjusting for chance. A value of 0.9979 shows near-perfect agreement, meaning the model's predictions are almost perfectly aligned with reality, beyond random chance.

  • Hamming Loss: 0.0012

    Hamming Loss measures the proportion of incorrect labels out of the total number of labels. A low value of 0.0012 means that only a few predictions are incorrect, indicating the model makes very few mistakes in its predictions.

Model Details

  • Layers: 3 layers, 128 neurons
    • The input layer, which has a number of neurons equal to the number of features in the dataset, serves as the first point of contact for the input data.
    • The hidden layer, which contains 128 neurons, processes the data and captures complex patterns. The number of neurons here is chosen to balance model complexity and computational efficiency.
    • The output layer, which has a number of neurons corresponding to the number of classes, produces the final predictions (e.g., classification probabilities). This layer outputs the processed data in a form that aligns with the task at hand (e.g., classification results).

  • Activation: ReLU

    ReLU (Rectified Linear Unit) is used as the activation function in the hidden layers to introduce non-linearity into the model. This function transforms any negative values to zero and passes positive values unchanged, allowing the model to learn complex patterns and relationships in the data.

  • Optimizer: Adam

    Adam (Adaptive Moment Estimation) is an optimization algorithm that combines the advantages of both the Adagrad and RMSProp algorithms. It adapts the learning rate for each parameter during training, helping the model converge more efficiently and avoid overshooting the optimal solution. This leads to faster convergence and better performance, especially in large and complex datasets.

  • Loss Function: CrossEntropy

    CrossEntropy is the chosen loss function for this classification task. It compares the predicted class probabilities with the actual class labels, calculating the difference (or "loss"). The function penalizes the model more heavily for being confident in the wrong predictions, guiding it to adjust its weights during training in order to minimize the error.

  • Epochs: 10

    The model was trained over 10 epochs, meaning that the entire training dataset was processed 10 times.

  • Batch Size: 32

    A batch size of 32 means that during training, the model processes 32 data samples before updating its internal parameters (weights). Smaller batch sizes can offer more frequent updates but may be noisier, while larger batch sizes provide more stable updates but require more memory.

  • Class Weight: Weight for normal traffic set to 5.0 to avoid False Positives

    To address the potential class imbalance in the dataset a higher weight of 5.0 is assigned to the "normal traffic" class. This encourages the model to focus more on correctly identifying instances of normal traffic, which helps reduce the occurrence of false positives and true negatives.

  • Training Time: ~60 minutes

    The model took approximately one hour to train on the provided dataset.
